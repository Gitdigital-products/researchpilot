# CipherForge Governance

## Decision-Making

Lazy consensus model with cryptography expert oversight.
- Non-controversial changes: 72-hour review window
- Crypto-related changes: Must have cryptography reviewer approval
- Releases: 5-day RC period + maintainer vote

## Roles
- **Project Lead**: Strategic direction
- **Cryptography Reviewers**: All crypto code changes require their approval
- **Maintainers**: PR review, governance
- **Contributors**: PRs, issues

## Adding Maintainers
2/3 maintainer vote + cryptography reviewer recommendation.
